Prerequisites & Requirements

System Requirements

  • Operating System: Linux (Ubuntu 20.04+), macOS 10.15+, or Windows 10+ with WSL2
  • Memory: Minimum 8GB RAM (16GB+ recommended for production)
  • Storage: 50GB+ free disk space
  • CPU: 4+ cores (8+ cores recommended)
  • Network: Stable internet connection for data streaming

Required Software

  • Docker (20.10+) and Docker Compose (2.0+)
  • Git (2.30+)
  • Node.js (16+) and npm (8+)
  • R (4.0+) with development tools
  • Python (3.8+) for quantum computing components

Optional Components

  • Kubernetes (1.20+) for scalable deployment
  • Terraform (1.0+) for cloud infrastructure
  • API keys for external data sources (Alpha Vantage, Polygon.io)
  • Quantum computing API tokens (IBM Quantum, IonQ)
Note: All software dependencies are automatically installed when using Docker deployment. Manual installation is only required for development setup.

Infrastructure Deployment

Clone the Repository

First, clone the NEURICX repository and navigate to the deployment directory:

bash 
git clone https://github.com/your-org/neuricx.git
cd neuricx/deploy

Configure Environment

Set up your environment variables by creating a .env file:

bash 
# Copy the example environment file
cp .env.example .env

# Edit the environment file with your configuration
nano .env
Important: Update the API keys and database passwords in the .env file before deployment.

Deploy Infrastructure

Use the automated deployment script for quick setup:

bash 
# Make the script executable
chmod +x deploy.sh

# Deploy for development
./deploy.sh deploy development

# Deploy for production
./deploy.sh deploy production your-domain.com true

Or deploy manually with Docker Compose:

bash 
# Start all services
docker-compose up -d

# Check service status
docker-compose ps

# View logs
docker-compose logs -f

Verify Deployment

Check that all services are running correctly:

bash 
# Check API health
curl http://localhost:8000/health

# Check dashboard
curl http://localhost:3000

# Check database connection
docker-compose exec postgres pg_isready

Cloud Deployment with Terraform

For production cloud deployment, use the provided Terraform configurations:

bash 
cd terraform

# Initialize Terraform
terraform init

# Plan deployment (AWS example)
terraform plan -var="cloud_provider=aws" -var="environment=production"

# Apply configuration
terraform apply

Launch Capabilities

Using the Web Platform

Access the NEURICX platform at platform.html to launch capabilities through the web interface.

Economic Simulation

Create and run multi-agent economic simulations:

r 
library(NEURICX)

# Create economy
economy <- create_neuricx_economy(
  n_agents = 1000,
  symbols = c("AAPL", "MSFT", "GOOGL"),
  network_density = 0.05
)

# Run simulation
economy <- run_neuricx_simulation(economy, n_steps = 250)

Real-time Data Streaming

Initialize live market data streaming:

r 
# Initialize stream
stream_config <- initialize_realtime_stream(
  symbols = c("AAPL", "MSFT", "GOOGL", "TSLA"),
  update_frequency = 60
)

# Start streaming
stream_config <- start_realtime_stream(stream_config)

Policy Analysis

Run comprehensive policy intervention analysis:

r 
# Monetary policy analysis
monetary_analysis <- analyze_monetary_policy(
  economy, 
  interest_rate_change = 25,
  intervention_timing = 50,
  duration = 25
)

Crisis Prediction

Predict economic crises using ensemble ML models:

r 
# Crisis prediction
crisis_prediction <- predict_economic_crisis(
  economy,
  prediction_horizon = 25,
  ensemble_models = c("neural_network", "random_forest", "xgboost", "lstm")
)

Quantum Optimization

Leverage quantum computing for advanced optimization:

r 
# Initialize quantum environment
quantum_env <- initialize_quantum_environment(
  backend = "simulator",
  quantum_config = list(n_qubits = 20)
)

# Quantum portfolio optimization
portfolio_result <- quantum_portfolio_optimization(
  economy,
  quantum_env,
  risk_tolerance = 0.5
)

API Usage Examples

Alternatively, use the REST API for programmatic access:

bash 
# Create economy
curl -X POST "http://localhost:8000/economy/create" \
  -H "Content-Type: application/json" \
  -d '{"n_agents": 1000, "symbols": ["AAPL", "MSFT", "GOOGL"]}'

# Start simulation
curl -X POST "http://localhost:8000/simulation/run" \
  -H "Content-Type: application/json" \
  -d '{"economy_id": "economy_123", "n_steps": 250}'

# Get results
curl "http://localhost:8000/simulation/simulation_456/results"

Troubleshooting

Common Issues

Services Not Starting

Problem: Docker containers fail to start

Solutions:

  • Check Docker daemon is running: docker --version
  • Verify sufficient disk space: df -h
  • Check port conflicts: netstat -tulpn | grep :8000
  • Review container logs: docker-compose logs [service_name]

Database Connection Issues

Problem: Cannot connect to PostgreSQL

Solutions:

bash 
# Check database status
docker-compose exec postgres pg_isready

# Reset database
docker-compose down
docker volume rm neuricx_postgres-data
docker-compose up -d postgres

Memory Issues

Problem: Out of memory errors during large simulations

Solutions:

  • Reduce number of agents or simulation steps
  • Increase Docker memory limits
  • Use distributed computing for large-scale simulations
  • Monitor memory usage: docker stats

Debugging Commands

bash 
# Check all service status
docker-compose ps

# View logs for all services
docker-compose logs

# View logs for specific service
docker-compose logs -f neuricx-api

# Execute commands in container
docker-compose exec neuricx-api bash

# Check system resources
docker system df

# Clean up unused resources
docker system prune

Performance Optimization

  • Adjust container memory limits in docker-compose.yml
  • Use SSD storage for database volumes
  • Enable Redis caching for frequently accessed data
  • Configure connection pooling for database connections
  • Use CDN for static web assets
